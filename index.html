Deepfake Recognizer was built to address key challenges in modern media verification, including privacy risks, server-side processing overhead, and delayed detection. By leveraging a browser-native architecture, the platform delivers secure, real-time deepfake analysis without transmitting user data externally, creating a more trustworthy and accessible solution for digital content authentication.

How It Works

When you upload a file, the entire analysis pipeline runs inside your browser using only built-in Web APIs. No model weights. No external calls. No database lookups.

01
File Load
File API + createObjectURL() loads media entirely in browser memory
02
Pixel Extract
Invisible <canvas> draws frames; getImageData() reads raw RGB pixels
03
Feature Analysis
12+ statistical metrics computed: noise, entropy, flicker, compression patterns
04
Metadata Scan
Raw bytes searched for AI tool signatures: Stable Diffusion, Midjourney, etc.
05
Risk Score
Weighted heuristics combine all signals into a 0–100% risk score with explanations
// detection signals

What the Detector Looks For

🔬
Texture Noise Analysis
Real camera photos contain tiny random sensor noise even on smooth skin. AI generators produce unnaturally "plastic" surfaces. The app measures high-frequency pixel variation vs. edge strength to detect over-smoothing.
📦
Compression Mismatch
JPEG saves photos in consistent 8×8 blocks. AI-generated images have different compression pipelines, creating inconsistent block patterns. The detector samples blocks and flags unnatural jumps between adjacent areas.
💡
Lighting Imbalance
Checks if one half of the image is significantly brighter than the other. A weak signal on its own, but meaningful when combined with other synthetic indicators like smoothness and color regularity.
🎨
Color Regularity / Entropy
Measures histogram entropy of color distribution. Real photos have high entropy from natural variation. AI-generated images often have suspiciously regular or unnaturally balanced color distributions.
🎬
Temporal Flicker (Video)
Samples 10–28 frames and measures luma/saturation jumps between consecutive frames. AI videos often show random brightness flickers that don't correspond to actual motion — a key deepfake signal.
🏷️
Metadata Signature Scan
Searches raw file bytes and EXIF data for known AI tool strings: "Stable Diffusion", "Midjourney", generation prompts, or absence of typical camera make/model data that real photos carry.
// risk scoring

Understanding the Risk Score

0% — Likely Real ~50% — Inconclusive 100% — Strong AI Signals
0–37%
Low signals — no strong evidence of AI generation found
38–67%
Inconclusive — some signals present, investigate further
68–100%
Strong signals — high probability of AI generation detected

⚠️ Important: A low score is NOT proof of authenticity. A high score is NOT forensic proof of fakery. The score is a statistical indicator — always cross-verify using reverse image search, source credibility, and human judgment.

// privacy architecture

Privacy by Design

NEVER HAPPENS

File sent to server

Your photo or video is never transmitted over the network. Not to the website host, not to Netlify, not to anyone. File loading uses the browser's File API entirely in local memory.

NEVER HAPPENS

Data stored persistently

The app uses no localStorage, IndexedDB, or cookies for media. Only lightweight report metadata (filename, score, timestamp) is kept in a JS array — max 6 entries — and disappears on page refresh.

NEVER HAPPENS

App owner sees your file

All processing happens inside your browser's JavaScript sandbox. The website only serves static HTML/CSS/JS files — like a calculator app, the creator cannot see what you type.

ALWAYS TRUE

Works offline after load

After the initial page load, you can disconnect from the internet and analysis still works fully. No external APIs, model servers, or databases are called during detection.

ALWAYS TRUE

File recovered after close?

Impossible. The temporary object URL and canvas pixel data are released when you close or refresh the tab. Browser memory is not accessible to other websites after the session ends.

ALWAYS TRUE

Can run from local folder

You can save the entire project folder and open index.html directly from your computer — no internet connection ever required. The website exists only for easy public access.

// known limitations

Limitations & False Positives

Transparency is a core design principle. Here's exactly what the detector struggles with:

High-Quality Deepfakes
Modern generative models (newer Stable Diffusion, Sora-style video) produce fewer statistical artifacts. Very well-made fakes can receive low scores — a low score does NOT guarantee authenticity.
Heavy Filters & Editing
AI beauty filters, heavy denoising, or multiple WhatsApp re-compressions alter the very signals the detector measures — real photos can appear suspicious after heavy processing.
Screenshots & Re-uploads
Taking a screenshot adds new compression layers and screen noise that mask original generation artifacts. Always analyze the original file, not a screenshot of it.
Novel Generation Techniques
Entirely new AI methods that don't match current heuristics will evade detection. The rule-based approach has fixed blind spots unlike an ML classifier that can be retrained.
Semantic Lies
A video can be technically real but contain false claims in what people are saying. This detector only checks visual authenticity, not factual accuracy of the content.
No Ground Truth Database
The detector uses general statistical rules, not a database of known fakes. It cannot match against specific previously identified deepfakes — it identifies patterns, not instances.
// ethical use

Responsible Use Guidelines

🔍

Check permission before analyzing someone's media

The tool never uploads files — but the act of analyzing private images of others without consent is still ethically wrong. Only analyze media you have permission to examine, or public content being verified for misinformation.

⚖️

Never use a score to make accusations alone

The score is one data point, not forensic proof. A mid-range or high score can result from heavy editing of a real photo. Always gather multiple forms of evidence before drawing conclusions about any person.

🚫

Don't share suspected deepfakes before verifying

Sharing unverified content, even with good intentions, amplifies harm. Use this tool + reverse image search + source credibility + human judgment together before deciding whether or how to share anything.

🎓

Found a deepfake of someone from school?

1) Do not share it further. 2) Save the original URL. 3) Report it on the platform (Instagram/TikTok have deepfake-specific reporting). 4) Tell a trusted teacher or parent. 5) Involve school or law enforcement if serious. Present the detector result as supporting information, not sole proof.

🛡️

Technology is only one layer of defense

Media literacy, platform policies, and critical thinking are equally important. This tool helps spot statistical clues — but humans must still verify sources, assess context, and use common sense. No automated detector replaces human judgment.
